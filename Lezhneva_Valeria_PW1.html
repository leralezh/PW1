a.On 28 October 2014, HTML5 was released as a W3C Recommendation, bringing the specification process to completion.The main goals behind HTML5 were multimedia support, readability and consistency, backward compatibility, interoperability, complex web applications.

b.The HTML5 specification, initially developed by the WHATWG, was the fifth major version of HTML. It became a retired W3C recommendation.The current standard, known as the HTML Living Standard, is maintained by the WHATWG and is continuously updated, while the W3C produces “snapshots” based on the WHATWG spec for stability.

c.The minimal HTML page code consists of the following essential elements:
  • Document type declaration (DOCTYPE)
  • HTML structure
  • Head section
  • Body content
d.Semantic HTML focuses on reinforcing the meaning of information in web pages and applications, rather than merely defining their presentation. It is processed by browsers and other user agents, while CSS handles its visual styling for human users. In contrast, presentational markup directly specifies how content should look, but it is now considered deprecated due to the push for separating content and presentation.

e.To check browser support for particular tag or attribute we can validate HTML in our pages using tools like the W3C Validator. It ensures compliance with HTML5 standards and highlights any deprecated elements or attributes that should no longer be used

f.The HTML5 video element, introduced by the WHATWG, is the widely supported video playback technology in modern browsers. It replaced the proprietary Adobe Flash plugin. The officially supported video formats are MP4, WebM, and Ogg. If other formats are used, compatibility issues may arise, as browsers adhere to these specified formats

For audio, the HTML standard supports MP3, WAV, and Ogg formats. Using these ensures consistent playback across browsers, while other formats may not be universally recognized
